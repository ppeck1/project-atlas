# Project Atlas v1.2

**Local-first personal project management.**  
*What am I carrying, what is blocked, what needs action, what goes to my phone?*

Flutter desktop app backed by local SQLite (Drift). No cloud. No sync.

---

## Quick Start

```powershell
# First time — or after any schema/source change:
.\launch.ps1 -Full

# Normal daily launch (skips unnecessary steps):
.\launch.ps1
```

Or manually:
```powershell
flutter pub get
dart run build_runner build
flutter run -d windows
```

> **If the app crashes on startup with "duplicate column name":** a prior migration
> was partially applied. Delete the database file, then relaunch. All local data
> will be lost. Migration v4+ is now defensive, so this should not recur.
> ```
> %APPDATA%\project_atlas\project_atlas.sqlite
> ```

## Ollama (local AI — optional)

AI output is **always shown for review before saving.** Nothing is auto-applied.

```sh
# Install from https://ollama.ai, then pull a model:
ollama pull mistral
# Ollama listens on http://localhost:11434 by default
```

Configure host + model in Settings → Ollama, then use AI buttons in Review,
Export, or the task edit sheet.

---

## Telegram (outbound only — optional)

1. Create a bot via [@BotFather](https://t.me/botfather) and copy the token
2. Get your chat ID: `https://api.telegram.org/bot<TOKEN>/getUpdates`
3. Settings → Telegram → enter token + chat ID → Test Connection
4. Export → "Send Today's List to Telegram"

All user text is HTML-escaped before sending.  
The bot token is stored in the local SQLite file (plaintext on disk).  
Do not commit the database file. It lives in `%APPDATA%`, outside the repo.

## Database

- Engine: SQLite via Drift (plain NativeDatabase, WAL)
- Location: `%APPDATA%\project_atlas\project_atlas.sqlite`
- Encryption: **not currently enabled** — SQLCipher is a planned step
- Schema: v4 — see `VARIABLE_MAP.md` for full migration history

## Architecture

```
lib/
  app/           App widget, theme, router
  db/            Drift tables + AppDb  ← app_db.g.dart generated by build_runner
  services/      OllamaService, TelegramService
  features/      projects / dashboard / today / work / review / export / governance / settings
  shared/
    models/      AppState (ChangeNotifier wrapping AppDb)
    widgets/     Navigation shell, dialogs
```

See **`VARIABLE_MAP.md`** for every table column, AppMeta key, service field, and data flow.

## Development

```powershell
.\launch.ps1          # smart launch — only rebuilds what changed
.\launch.ps1 -Full    # force pub get + build_runner + run
.\launch.ps1 -Build   # codegen only, no run
.\launch.ps1 -Clean   # flutter clean + full rebuild + run
```

After changing `tables.dart` or `app_db.dart`, always re-run build_runner before launching.

**Never commit:** `lib/db/app_db.g.dart`, `.dart_tool/`, `build/`, `*.bak`
